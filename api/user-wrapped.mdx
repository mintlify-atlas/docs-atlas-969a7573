---
title: User Wrapped
description: Generate year-in-review analytics for GitHub users
---

<Note>
Endpoint: `GET /api/wrapped/user`
</Note>

Generate comprehensive year-in-review analytics for any GitHub user, including contribution patterns, top repositories, activity streaks, and language statistics.

## Request

### Query Parameters

<ParamField query="username" type="string" required>
  GitHub username
  
  Example: `torvalds`, `gaearon`, `tj`
</ParamField>

<ParamField query="year" type="number" default="Previous year">
  Year to generate wrapped for (2008 - current year)
  
  Example: `2024`
</ParamField>

<ParamField query="accessToken" type="string">
  GitHub personal access token for private activity
  
  **Note**: Required to include private repository contributions. When provided, caching is disabled.
</ParamField>

### Validation Rules

- `username`: Must be at least 1 character
- `year`: Integer between 2008 and current year

## Examples

### Basic Request

<CodeGroup>
```bash cURL
curl "https://your-domain.com/api/wrapped/user?username=torvalds&year=2024"
```

```javascript JavaScript
const response = await fetch(
  'https://your-domain.com/api/wrapped/user?username=torvalds&year=2024'
);
const { data, cached } = await response.json();

console.log(`Total commits: ${data.overview.totalCommits}`);
console.log(`Longest streak: ${data.streak.longest} days`);
```

```python Python
import requests

response = requests.get(
    'https://your-domain.com/api/wrapped/user',
    params={'username': 'torvalds', 'year': 2024}
)

data = response.json()['data']
print(f"Total PRs: {data['overview']['totalPRs']}")
print(f"Active days: {data['streak']['totalActiveDays']}")
```
</CodeGroup>

### Include Private Activity

<CodeGroup>
```bash cURL
curl "https://your-domain.com/api/wrapped/user?username=myusername&year=2024&accessToken=ghp_xxxxx"

const response = await fetch(
  'https://your-domain.com/api/wrapped/user?' + new URLSearchParams({

    accessToken: process.env.GITHUB_TOKEN
  })

const { data } = await response.json();
```
</CodeGroup>

## Rate Limiting

The endpoint checks GitHub API rate limits before processing:

- Requires at least **50 remaining requests** to proceed (higher than repository wrapped)
- Returns 429 status if rate limit is too low
- User data requires more API calls than repository data

## Caching Behavior

- Public user data is cached for **24 hours**
- Private user data is **never cached**
- Requests with `accessToken` parameter bypass cache
- Cache key format: `wrapped:user:{username}:{year}`

## Privacy Considerations

- Public users can be queried by anyone
- Private contributions require authentication with appropriate scopes
- Private repository activity is only visible to authenticated users with access
