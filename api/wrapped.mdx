---
title: Repository Wrapped
description: Generate year-in-review analytics for GitHub repositories
---

<Note>
Endpoint: `GET /api/wrapped`
</Note>

Generate comprehensive year-in-review analytics for any GitHub repository, including contribution statistics, language breakdown, release history, and community growth metrics.

## Request

### Query Parameters

<ParamField query="owner" type="string" required>
  Repository owner (username or organization name)
  
  Example: `vercel`, `facebook`, `torvalds`
</ParamField>

<ParamField query="repo" type="string" required>
  Repository name
  
  Example: `next.js`, `react`, `linux`
</ParamField>

<ParamField query="year" type="number" default="Previous year">
  Year to generate wrapped for (2008 - current year)
  
  Example: `2024`
</ParamField>

<ParamField query="accessToken" type="string">
  GitHub personal access token for private repositories
  
  **Note**: When provided, caching is disabled to protect private data.
</ParamField>

### Validation Rules

- `owner`: Must be at least 1 character
- `repo`: Must be at least 1 character  
- `year`: Integer between 2008 and current year
- `accessToken`: Optional, required for private repositories

## Examples

### Basic Request

<CodeGroup>
```bash cURL
curl "https://your-domain.com/api/wrapped?owner=vercel&repo=next.js&year=2024"
```

```javascript JavaScript
const response = await fetch(
  'https://your-domain.com/api/wrapped?owner=vercel&repo=next.js&year=2024'
);
const { data, cached } = await response.json();

console.log(`Total commits: ${data.activity.total}`);
console.log(`Cached response: ${cached}`);
```

```python Python
import requests

response = requests.get(
    'https://your-domain.com/api/wrapped',
    params={
        'owner': 'vercel',
        'repo': 'next.js',
        'year': 2024
    }
)

data = response.json()['data']
print(f"Stars gained: {data['community']['starsGained']}")
```
</CodeGroup>

## Rate Limiting

The endpoint checks GitHub API rate limits before processing:

- Requires at least **10 remaining requests** to proceed
- Returns 429 status if rate limit is too low
- Authenticated requests have higher limits (5,000/hour vs 60/hour)

## Caching Behavior

- Public repository data is cached for **24 hours**
- Private repository data is **never cached**
- Requests with `accessToken` parameter bypass cache
- Cache key format: `wrapped:{owner}:{repo}:{year}`
