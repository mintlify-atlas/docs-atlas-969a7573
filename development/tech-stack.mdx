## Caching & Performance

<CardGroup cols={2}>
  <Card title="Upstash Redis" icon="database" href="https://upstash.com">
    Serverless Redis for distributed caching with automatic scaling.
    
    **Package**: `@upstash/redis@^1.35.7`
    
    **Configuration**:
    - 24-hour TTL for wrapped data
    - Automatic fallback to in-memory cache
    - REST-based (no persistent connections)
  </Card>
  
  <Card title="In-Memory Cache" icon="memory">
    Map-based fallback cache for development and environments without Redis.
    
    **Implementation**: `lib/cache.ts`
    
    - Same 24-hour expiration
    - Type-safe cache entries
    - Automatic cleanup
  </Card>
</CardGroup>

## Environment Variables

<CodeGroup>
```bash Optional - GitHub API
GITHUB_TOKEN=ghp_your_token_here
# Increases rate limit from 60/hour to 5,000/hour
```

```bash Optional - Redis Cache
UPSTASH_REDIS_REST_URL=https://your-redis.upstash.io
UPSTASH_REDIS_REST_TOKEN=your_token_here
# Falls back to in-memory cache if not configured
```

```bash Required - Database (for auth)
DATABASE_URL=postgresql://user:pass@host:5432/db
# Required for better-auth session storage
```

```bash Required - Auth
BETTER_AUTH_SECRET=your_secret_here
BETTER_AUTH_URL=http://localhost:3000
# Or your production URL
```
</CodeGroup>

## Package Manager

The project uses **pnpm** for faster installs and better disk efficiency:

```bash
pnpm install    # Install dependencies
pnpm dev        # Start dev server
pnpm build      # Production build
pnpm start      # Start production server
pnpm lint       # Run ESLint
```

<Note>
  While pnpm is recommended, the project also works with npm or yarn.
</Note>
