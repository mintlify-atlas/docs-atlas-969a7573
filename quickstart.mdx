## Create Your First Wrapped

Get started with GitHub Wrapped in just a few clicks. No installation required - just visit the app and start exploring!

<Steps>
  <Step title="Visit GitHub Wrapped">
    Navigate to the GitHub Wrapped application in your browser. You'll see a clean interface with a search input and year selector.
    
    <Tip>
      The app works on any device - desktop, tablet, or mobile!
    </Tip>
  </Step>

  <Step title="Enter a Repository">
    In the repository input field, enter a GitHub repository in one of these formats:
    
    - **owner/repo** format: `facebook/react`
    - **Full GitHub URL**: `https://github.com/vercel/next.js`
    
    <Note>
      **Signed in users** can select from a dropdown of their own repositories, including private ones. Public repositories work for everyone!
    </Note>
    
    The app will validate the repository in real-time to ensure it exists and is accessible.
  </Step>

  <Step title="Select a Year">
    Choose any year from 2008 to the current year using the year selector dropdown. 
    
    Pick a year when the repository was active to see the most interesting data!
    
    <Tip>
      For most active repositories, the previous year or current year will have the most complete data.
    </Tip>
  </Step>

  <Step title="Generate Your Wrapped">
    Click the **GO** button to start generating your wrapped. 
    
    The app will:
    1. Validate your repository
    2. Fetch data from the GitHub API
    3. Calculate comprehensive analytics
    4. Generate your animated story
    
    <Info>
      **First-time generation**: 5-10 seconds for small-to-medium repos  
      **Cached results**: Under 3 seconds (cached for 24 hours)
    </Info>
  </Step>

  <Step title="Explore Your Story">
    Navigate through the slides using:
    
    - **Arrow keys** (← →) on desktop
    - **On-screen buttons** on any device
    - **Spacebar** to auto-play through all slides
    
    Each wrapped story includes 8-12 slides featuring:
    - Welcome screen with repository info
    - Total commits and basic stats
    - Top contributor spotlight
    - Community growth metrics
    - Busiest month analysis
    - Language breakdown
    - Major releases (if applicable)
    - Night owl and weekend warrior patterns
    - Complete year summary
    - Sharing options
  </Step>

  <Step title="Share Your Wrapped">
    On the final slide, you can:
    
    - **Share on Twitter** - Pre-filled tweet with your wrapped link
    - **Copy link** - Share the URL anywhere
    - **Generate a new wrapped** - Try another repository or year
    
    Every wrapped has a unique shareable URL like:  
    `/wrapped/owner/repo/2024`
  </Step>
</Steps>

## User vs Repository Wrapped

GitHub Wrapped supports two types of wrapped stories:

### Repository Wrapped

Analyzes a specific repository to show:
- Repository-wide commit activity
- All contributors and their impact
- Community growth (stars, forks, contributors)
- Release history and milestones
- Language statistics

**Use case**: Team retrospectives, open source project summaries, celebrating milestones

### User Wrapped

Focuses on an individual GitHub user to show:
- Personal contribution patterns
- Most active repositories
- Coding time preferences
- Language preferences
- Contribution streaks

**Use case**: Personal year-in-review, developer portfolios, team member spotlights

<Note>
  To generate a **User Wrapped**, enter just the username (not owner/repo) in the repository input field.
</Note>
